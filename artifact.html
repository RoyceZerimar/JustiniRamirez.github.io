SecureVault (Python)

SecureVault (Password Manager) Artifact
By Justin Ramirez

Briefly describe the artifact. What is it? When was it created?

The artifact I chose for my ePortfolio is my Password Manager project. I first created a very simple version about two years ago. The original program was a single-file Tkinter script that generated passwords and stored them in a JSON file for one user. It had no encryption, no database, and no real structure.

For this course, I rebuilt the project into a much more complete application called SecureVault. The new version uses multiple modules, real encryption, a SQLite database, and support for multiple users. It is a much more professional and secure version of the original idea.

Justify the inclusion of the artifact. Why did you select it? What skills does it showcase? How was it improved?

I selected this artifact because it represents my growth across all three capstone categories: Software Engineering, Algorithms & Data Structures, and Databases. It is also one of the few projects I fully built, used, and then completely redesigned from the ground up.

Software Engineering skills shown:

  • Reorganizing the project into a clean, modular structure
  • Removing outdated code and improving readability
  • Using constants and separating logic into modules
  • Following industry-standard practices for maintainability

Algorithms & Data Structures skills shown:

  • Designing secure authentication logic
  • Implementing encryption and decryption workflows
  • Managing user accounts and vault operations
  • Structuring the program so each module handles a specific responsibility

Database skills shown:

  • Replacing the old JSON file with a real SQLite database
  • Designing tables for users and password entries
  • Writing SQL queries to insert, update, and retrieve data
  • Connecting the database to the UI and backend logic
  • Ensuring encrypted data is stored safely and consistently

How the artifact was improved:

  • Converted from a single-file script into a full multi-file application
  • Added encryption for stored passwords
  • Added multi-user support
  • Added a SQLite database with proper schema design
  • Cleaned up the codebase and improved readability
  • Made the entire system more secure, scalable, and professional

Reflect on the enhancement process. What did you learn? What challenges did you face? How did you use feedback? Which outcomes were met?

Working on this artifact taught me how important proper structure, security, and data management are in real applications. Rebuilding the project forced me to think about how each part of the system connects the UI, the database, the encryption, and the user logic.

What I learned:

  • How to design cleaner, more organized code
  • How encryption works and how to apply it correctly
  • How to design and use a database instead of a simple file
  • How to build a project that supports multiple users
  • How to think like a software engineer instead of just writing code that “works”

Challenges I faced:

  • Figuring out which enhancements belonged in the milestone version and which belonged in the final version
  • Designing secure encryption that worked with the database
  • Migrating from JSON to SQLite without breaking the program
  • Debugging issues where the UI wasn’t pulling the right data
  • Making sure passwords saved and loaded correctly after logging out

How feedback was used:

  • Instructor feedback helped me focus on cleaner, more modular code
  • Feedback also helped me understand which enhancements fit each milestone
  • I used feedback to decide what improvements should wait for the final ePortfolio version
